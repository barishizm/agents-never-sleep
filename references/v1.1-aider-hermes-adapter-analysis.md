# ANS v1.1 adapter analysis — Aider + Hermes — and the v1.2+ agent survey

> Research 2026-06-28. Companion to `cross-platform-enforcement-design.md` (the original
> 6-platform matrix). This is the capability research + adapter design for the next two
> platforms (Aider, Hermes) and the prioritised plan for the rest. All file:line refs were
> verified against installed source. **Nothing here is live-smoked** — live verification per
> platform stays a manual Mes step (no platform but Claude is in `LIVE_VERIFIED`).

The three guarantees, as always: **deny_irreversible · never_stop · never_ask**. An adapter is
NATIVE on a guarantee only where the host gives a real interception seam; otherwise it is
🟡 soft-enforced (prose contract + a loud morning-report BLIND SPOT via `capabilities.py`).

A reframe that shapes the whole plan: **never_stop has two failure modes.**
(a) *turn ends early* → needs a stop hook that can VETO end-of-turn (rare; ≈Claude-only).
(b) *process exits with backlog incomplete* → handled by **ANS's own driver relaunching on the
`UE_RUN_INCOMPLETE` sentinel** — platform-independent, needs no host hook. Every one-shot headless
invocation (`aider --message`, `opencode run`, `amp -x`, `goose run`, …) is covered by (b). So
never_stop is *soft-via-relaunch* everywhere headless, and *native* only where a blocking stop hook
exists.

## 1. Aider (ticket `_future/10`) — platform #7, the first all-soft / wrapper adapter

**Subject:** Aider 0.86.2 (`/home/claude/.local/bin/aider`, pipx venv). Skill: `aider-coding-tool`.

**Headline:** Aider has **no hook / plugin / event / callback API of any kind** (verified — the only
`hook` hits in `args.py` are git's `--git-commit-verify`). There is **no pre-shell-execution seam to
deny on**, so Aider breaks the old invariant *"deny-irreversible works on every platform."* It is a
fundamentally different adapter SHAPE: a **wrapper / launch-preset + git-reversibility + prose**, not
a hook-payload dispatcher. All three cells are soft-enforced.

**Execution model.** Aider is one-shot in message mode: `BaseCoder.run(with_message=…)`
(`coders/base_coder.py:876`) calls `run_one(...)` once and returns — it never enters the interactive
`while True` loop. `main.py:1126` wires `--message`/`--message-file` then the process exits. **The ANS
loop sits entirely OUTSIDE aider**, driving it as a subprocess once per ticket.

**Launcher preset (the adapter's core):**
```bash
UE_UNATTENDED=1 UE_PLATFORM=aider \
aider --message-file .unattended/ticket-<id>.prompt \
      --yes-always --no-detect-urls \
      --no-suggest-shell-commands \      # kill the LLM-suggested-shell path
      --no-auto-test --no-auto-lint \    # never wire operator test/lint cmds
      <files...> < /dev/null             # EOF on any un-anticipated prompt -> clean exit
# ANS records pre-invocation HEAD SHA, runs the above, runs ITS OWN gate (tsc/tests),
# and on failure: git reset --hard <pre-SHA>  (ANS's revert, which the irreversible matcher
# deliberately does NOT block).
```

**Per-guarantee findings:**
- **deny_irreversible → 🟡 soft (genuine residual hole).** Two shell paths. (1) LLM-suggested shell
  (`handle_shell_commands`, `base_coder.py:2450`) auto-DECLINES under `--yes-always` (`io.py:866`:
  `explicit_yes_required` → returns `"n"`) and is killed by `--no-suggest-shell-commands`. ✅ (2)
  Operator `--test-cmd`/`--auto-test`/`--lint-cmd` and `/run` `/test` call `run_cmd`
  (`commands.py:983/1003` → `run_cmd.py:11` → `subprocess.Popen`) with **no confirm and no hook** — an
  un-interceptable shell path. 🔴 This is why deny-irreversible can't be NATIVE. Mitigation: the preset
  never sets those flags; ANS runs its own gate; git-revert recovers file changes; sandbox the run.
- **never_stop → 🟡 soft, structurally strong.** No stop event exists, but none is needed: aider is a
  one-shot subprocess; continuation is the outer ANS loop (failure mode (b) above). Do NOT model this
  as a stop-hook emit.
- **never_ask → 🟡 soft, robust by preset.** `--yes-always` auto-answers `confirm_ask`/`prompt_ask`;
  belt-and-suspenders `stdin < /dev/null` → any unanticipated prompt hits EOF → falls back to default
  (`io.py:961`) and `run()` catches `EOFError` and returns cleanly (`base_coder.py:884`). Aider
  **structurally cannot hang waiting for a human.**

**Reversibility leverage.** Aider auto-commits each change (`auto_commits=True`, `base_coder.py:308`).
ANS reverts to the **captured pre-invocation HEAD SHA**, never `HEAD~1` (one `--message` can emit
multiple commits — edit + a separate "Ran the linter" commit, `base_coder.py:1601`). Keep auto-commit
ON. Nuance: aider passes `--no-verify` by default (`--git-commit-verify` default `False`,
`args.py:494`) → repo pre-commit hooks are **skipped** by default (so a pre-commit `decide()` bridge
wouldn't even fire without flipping that flag — note for any future enhancement).

**Matrix row:** `"aider": {DENY_IRREVERSIBLE: DEGRADED, NEVER_STOP: DEGRADED, NEVER_ASK: DEGRADED}`
(first all-soft platform). Consider an aider-specific note so the report doesn't read never_stop/
never_ask as weak as a true missing-hook case — they're soft-but-structurally-strong.

**Adapter shape:** `hooks/platforms/aider/` is a **launch recipe + README**, not a `hooks.json`.
`enforce.py`/`_normalize`/`_emit` stay **untouched** for aider (no dispatcher event). The adapter =
launcher preset (never_ask + deny-prevention) + git-reversibility wrapper (deny-recovery) + prose
(SKILL.md, the residual cells) + the loud BLIND SPOTs.

**Hermetic tests (no live aider):** `test_capabilities` (the row, 3 blind-spot lines, not in
LIVE_VERIFIED); a new `test_aider_launcher` asserting the generated argv (`--yes-always`,
`--no-suggest-shell-commands`, auto-commits on, no `--test-cmd`, stdin=/dev/null); a reversibility
assertion that the wrapper reverts to the captured pre-SHA (throwaway temp git repo, never invoke
aider). No dispatcher test for aider (intentionally — comment it in `test_enforce_platforms.py`).

**`_HOOK_CONTRACT["aider"]`** = a version-pinned *behavioral* contract (drift risk is aider's
semantics, not a hook API), e.g. *"Aider 0.86.2 — NO hook API (wrapper adapter); relies on
--yes-always (io.py:866), auto-commit (base_coder.py:308), one-shot run (base_coder.py:876), and the
KNOWN HOLE that cmd_test/cmd_run bypass confirm (commands.py:983/1003). Re-verify on any aider
upgrade."*

**Blast / open risks:** (1) `--test-cmd`/`--auto-test`/`--lint-cmd` = highest risk (arbitrary shell, no
confirm, no hook) → preset must never set them, document as a hard prohibition. (2) `/run` `/test`
share that un-gated path. (3) aider needs its own LLM key → that spend is **outside** ANS billing (cost
note). (4) startup network (versioncheck/URL scrape) is not ANS-interceptable → `--no-detect-urls` +
sandbox egress.

## 2. Hermes (ticket `_future/11`) — the best fit after Claude; first in-process adapter

**Subject:** `hermes` (hermes-orch-beta, Mes's own orchestrator, `/opt/hermes-orch-beta/code/`).
**Modifiable** (unlike third-party tools) → we can add a real seam.

**Lead finding — the exact gap ANS closes.** Hermes has two human gates with opposite timeout
semantics:

| Gate | On timeout | File:line |
|---|---|---|
| **Approval** (dangerous shell) | fail-CLOSED → DENY ("Timeout - denying command") | `approval.py:718`, `1226` |
| **Clarify** (ambiguous decisions) | **fail-OPEN → invented consent** ("the user did not provide a response … use your best judgement and proceed") | `cli.py:8655-8664` |

The dangerous-command path is already safe; the **clarify path is the hallucinated-consent hole**
(default `clarify.timeout=120s`). ANS converts a `clarify` call into an explicit **DENY → PARK before
the timeout is ever reached**, on every surface. That is the structural guard never-ASK was built for.

**Execution model.** Standard OpenAI tool-loop. Production `_invoke_tool` (`run_agent.py:9290`) routes
a few inline tools (`todo`/`memory`/`clarify`/`delegate_task`), the rest via
`handle_function_call → registry.dispatch`. A turn ends on **no tool calls** (`agent_loop.py:489`).
There is **no native Stop hook** that can veto end-of-turn (`on_session_end` is observer-only,
`plugins.py:88`).

**The decisive fact:** the **`pre_tool_call` plugin hook fires BEFORE the clarify/inline special-cases
on all dispatch paths** (`run_agent.py:9299` concurrent / `9815` sequential / `model_tools.py:687`
eval). A hook returning `{"action":"block","message":…}` short-circuits the tool
(`plugins.py:1172-1208`, single-fire). This is a **real native deny + native never-ASK seam.**

**Matrix row:** `"hermes": {DENY_IRREVERSIBLE: NATIVE, NEVER_STOP: DEGRADED, NEVER_ASK: NATIVE}`
- **deny_irreversible NATIVE** — `pre_tool_call` blocks any command-bearing tool via `decide()`.
- **never_ask NATIVE** — same hook denies `clarify` *before* the timeout/fabrication branch. Hermes
  is only the **2nd platform after Claude** with native never-ASK, because `clarify` is a real askable
  tool behind a real pre-hook.
- **never_stop 🟡 soft → NATIVE-via-patch** — no veto hook today; upgradeable by an in-tree edit at
  the no-tool-calls branch: if the run-incomplete sentinel exists, re-inject a "continue with the next
  ticket" message instead of finishing (bounded by the existing `_STOP_LOOP_CAP`). Ship soft + blind-
  spot first; stage the patch separately.

**Adapter shape — ANS's first NATIVE IN-PROCESS adapter (NOT the subprocess dispatcher).**
```
~/.hermes/plugins/agents-never-sleep/
  plugin.yaml   # provides_hooks: [pre_tool_call]
  __init__.py   # register(ctx): ctx.register_hook("pre_tool_call", _ans_pre_tool)
```
```python
def _ans_pre_tool(tool_name, args, **kw):
    from agents_never_sleep.enforcement import decide, Action
    if not _unattended():                          # honour UE_UNATTENDED / CLAUDE_UNATTENDED
        return None
    command = args.get("command") or args.get("code") or ""
    d = decide("pre_tool", tool_name=tool_name, command=command)
    return {"action": "block", "message": d.reason} if d.action == Action.DENY else None
```
Reuses **only `decide()`** from the shared core — `_normalize`/`_emit_*` do not apply. never-ASK
needs `"clarify"` in the ask-set: simplest is to add it to the shared `_ASK_TOOLS`
(`enforcement.py:25`) globally (harmless — no other platform exposes a tool named `clarify`); record
the decision. never_stop, when patched, is an in-tree edit, not a plugin hook.

**clarify → DENY/PARK.** Primary (no patch, ships in the plugin): the hook returns the ASK-deny PARK
directive for `tool_name=="clarify"`, firing before the 120s wait / the `cli.py:8661` fabrication / the
gateway "not available" error. The fail-open timeout becomes moot. Secondary (optional patch): also
make `clarify_tool` timeout return a PARK directive, for any future path that bypasses the hook.

**Hermetic tests (`acceptance/test_enforce_hermes.py`, no live Hermes — `systemctl --user` is
unreachable from the shell):** deny on `terminal{command:"git push --force"}`; never-ASK on
`clarify{question:…}`; benign-allow on `ls -la`; inert when not unattended; (when patched) never-stop
no-tool-calls branch capped by `_STOP_LOOP_CAP`; capabilities row + degradation/verification notes.

**`_HOOK_CONTRACT["hermes"]`** = *"Hermes pre_tool_call plugin hook (register_hook +
get_pre_tool_call_block_message {'action':'block'}) — 2026-06 in-tree contract."* Drift is
controllable (Mes's own code).

**Open risks:** (1) **Plugin must be in `config.yaml` `plugins.enabled`** (`plugins.py:740`) — opt-in;
install docs must add it + the run must verify it loaded (assert a startup deny fires). (2)
**never-ASK on the gateway is config-dependent** — `clarify` is excluded from the
`openai-api-server`/`editor` distributions (`toolsets.py:305,324`); the gateway resolves tools per
config, so never-ASK lands only when the session exposes `clarify`. CLI fail-open is always reachable
(the value story holds there). Record as OPEN: confirm the actual unattended Hermes config exposes
clarify. (3) never_stop soft until patched. (4) the `_ASK_TOOLS` global edit touches all platforms
(deliberate, low blast). **Hermes stays OUT of `LIVE_VERIFIED`** until Mes runs the smoke-test.

## 3. v1.2+ survey — the other clients (plan, not built)

Decisive question per agent: does it have a **pre-tool hook that sees the command string** (deny +
never-ASK) and a **headless run mode**? never_stop = soft-via-relaunch everywhere; native only with a
blocking stop hook.

| Agent | Headless | Pre-tool hook (sees cmd?) | Stop veto? | Predicted row (deny/stop/ask) | Shape | OSS? | Priority |
|---|---|---|---|---|---|---|---|
| **opencode** (sst) | `opencode run` | ✅ `tool.execute.before` (throws to deny) | 🟡 `session.idle` observe-only | native / soft-relaunch / native | JS plugin → shells `enforce` | ✅ MIT | **#1 HIGH** |
| **Crush** (charm) | yes; `--yolo` | ✅ shell hook **before permission check** | 🟡 not documented | native / soft-relaunch / native | shell hook ≈ Claude bash-hook | 🟡 FSL-1.1 | **#2 HIGH** |
| **Cline** | `cline -p`; `--yolo` | 🟡 hooks/SDK — deny seam UNCONFIRMED | 🟡 unconfirmed | ?/soft-relaunch/native | JS plugin via `@cline/sdk` | ✅ OSS | MED (verify) |
| **Amp** (Sourcegraph) | `amp -x` | ✅ `tool.call` (reject-and-continue) | 🟡 `agent.end` veto unconfirmed | native / soft-relaunch / native | JS/shell hook | 🔴 closed | MED |
| **Continue** (`cn`) | `cn -p` | 🟡 declarative perms.yaml only | 🔴 none | soft / soft-relaunch / native | declarative config | ✅ OSS | MED-LOW |
| **Goose** (Block) | `goose run`; `GOOSE_MODE=auto` | 🔴 per-TOOL gate — **can't deny by command content** | 🔴 none | soft / soft-relaunch / native | tool-perms + sandbox | ✅ Apache | MED-LOW |
| **OpenHands** | headless CLI | 🟡 SDK security-analyzer; CLI auto-approves | 🔴 none | soft / soft-relaunch / native | Python SDK integration | ✅ OSS | LOW-MED |
| **Kilo** | `--auto` | maybe inherits opencode engine | inherits opencode | likely = opencode | maybe covered by opencode adapter | ✅ OSS | LOW (verify) |
| **Roo** | — | — | — | — | — | archived 2026-05 | OUT |
| **Zed** | host (ACP) | n/a | n/a | — | — | Apache | OUT (host, not target) |

**Roadmap:** v1.2 = **opencode + Crush** (best seam + cheapest reuse); publish the opencode adapter as
an npm plugin and submit to `awesome-opencode` (discoverability — Tier-2). v1.3 = **Cline + Amp** once
their pre-tool-deny / stop-veto / OSS questions are confirmed. **Deprioritise** Goose (weak deny seam —
gates the whole shell tool, not the dangerous command), Continue (declarative), OpenHands (SDK-only),
Kilo (verify opencode-engine reuse). Treat every never_stop cell as soft-via-relaunch unless a blocking
stop hook is proven (consistent with how the matrix already treats Cursor/Windsurf).

**Discoverability:** opencode has a public npm plugin ecosystem (`@opencode-ai/plugin`) + an
`awesome-opencode` list → real Tier-2 win and reinforces opencode as #1. Cross-cutting:
`awesome-cli-coding-agents`, The AI Agent Index — an "ANS-compatible" entry aids discovery regardless of
per-agent adapter status.

   README + the design doc; bump SEMVER as a 1.1 minor (additive — new platforms, no API change).
